# 施設基準の構造

## 2022年7月現在の文法

### 告示の文法

```
public class 施設基準告示読込 extends BaseParser {

    public 施設基準告示読込() {
        add(施設基準等.value);
        add(漢数字第.value);
        add(施設基準別表.value);
        add(漢数字別表第.value);
        add(漢数字.value);
        add(数字括弧.value);
        add(カナ.value);
        add(数字丸.value);
        add(数字.value);
        add(漢数字括弧.value);
        add(StencilParagraph.value);
    }
}
```


### 通知の文法

```
public class 施設基準通知読込 extends BaseParser {

    public 施設基準通知読込() {
        add(章.value);
        add(数字第.value);
        add(数字の.value);
        add(数字括弧.value);
        add(カナ.value);
        add(カナ括弧.value);
//      add(英小文字.value);
        add(StencilParagraph.value);
    }
}
```

「数字の」は「１」および「１の２」には対応しているが、「１の２の２」には対応していない。
これが出現した場合は「１の３」にテキストを変更している。

```
public class 数字の extends StencilOrdered {

    public static final 数字の value = new 数字の();
    
    private 数字の() {
        super("数字の", MARGIN_PAT +
            "(?<" + NUMBER + ">" + "(?<" + ORDER + ">" + 数字n + "(の" + 数字n + ")?" + "))" +
            SEPARATOR_PAT + HEADER_PAT + "$");
        addCandidateParent(数字第.value);
    }
    .....
}
```
